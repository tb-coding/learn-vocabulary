# Spec-Kit Documentation

## What is Spec-Driven Development?

Spec-Driven Development flips the script on traditional software development. For decades, code has been king — specifications were just scaffolding we built and discarded once the "real work" of coding began. Spec-Driven Development changes this: specifications become executable, directly generating working implementations rather than just guiding them.

## Core Philosophy

Spec-Driven Development is a structured process that emphasizes:

- Intent-driven development where specifications define the "what" before the "how"
- Rich specification creation using guardrails and organizational principles
- Multi-step refinement rather than one-shot code generation from prompts
- Heavy reliance on advanced AI model capabilities for specification interpretation

## Development Phases

### Phase 0-to-1 Development ("Greenfield")
- Start with high-level requirements
- Generate specifications
- Plan implementation steps
- Build production-ready applications

### Phase Creative Exploration
- Explore diverse solutions
- Support multiple technology stacks & architectures
- Experiment with UX patterns

### Phase Iterative Enhancement ("Brownfield")
- Add features iteratively
- Modernize legacy systems
- Adapt processes

## Workflow Steps

1. **Create Constitution** - Define project governing principles
   ```
   /speckit.constitution Create project guidelines...
   ```

2. **Initialize Project** - Set up project structure
   ```
   specify init <PROJECT_NAME> --ai claude
   ```

3. **Generate Specifications** - Create detailed specs
   ```
   /speckit.spec Generate specifications for feature...
   ```

4. **Implement** - Build based on specifications
   ```
   /speckit.implement Build the feature according to spec...
   ```

5. **Test & Validate** - Ensure quality
   ```
   /speckit.test Run tests for...
   ```

## Key Commands

- `specify init` - Initialize a new project
- `specify check` - Check installed tools
- `/speckit.constitution` - Create project constitution
- `/speckit.spec` - Generate specifications
- `/speckit.implement` - Implement features
- `/speckit.test` - Test implementations
